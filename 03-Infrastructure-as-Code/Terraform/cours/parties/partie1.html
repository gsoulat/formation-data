📚 Introduction à Terraform

Qu'est-ce que Terraform ?

Terraform est un outil open-source d'Infrastructure as Code (IaC) développé par HashiCorp. Il permet de définir, provisionner et gérer l'infrastructure cloud de manière déclarative et automatisée.

💡 Infrastructure as Code (IaC)

L'IaC est la pratique de gérer et provisionner l'infrastructure via du code plutôt que par des processus manuels. Cela apporte versioning, réutilisabilité, automatisation et cohérence.

Pourquoi Terraform ?

  • Multi-cloud : Fonctionne avec Azure, AWS, GCP, et 100+ providers
  • Déclaratif : Vous décrivez l'état désiré, Terraform s'occupe du reste
  • Immutable : Infrastructure immuable et prévisible
  • Plan avant exécution : Preview des changements avant application
  • Gestion d'état : Tracking de l'infrastructure existante
  • Modules réutilisables : Composants d'infrastructure partagés
  • Collaboration : Versioning avec Git, CI/CD integration

Terraform vs Autres Outils

Outil Type Approche Cas d'usage
Terraform IaC Déclaratif, Multi-cloud Provisioning infrastructure
ARM Templates IaC Déclaratif, Azure only Provisioning Azure
Bicep IaC Déclaratif, Azure only ARM simplifié
Ansible Configuration Procédural Configuration serveurs
Azure CLI Command-line Impératif Scripts et automation

Concepts Clés

  • Provider : Plugin pour interagir avec un cloud provider (Azure, AWS, etc.)
  • Resource : Composant d'infrastructure (VM, VNet, Storage, etc.)
  • State : Fichier trackant l'état actuel de l'infrastructure
  • Plan : Preview des changements à appliquer
  • Apply : Exécution des changements
  • Module : Collection réutilisable de ressources
  • Variable : Paramètre d'entrée pour rendre le code flexible
  • Output : Valeur exportée après déploiement
