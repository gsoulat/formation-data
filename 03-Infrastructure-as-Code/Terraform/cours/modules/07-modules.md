## üìå Versioning des Modules

### Pourquoi Versionner ?

- ‚úÖ Stabilit√© : Emp√™che les changements inattendus
- ‚úÖ Reproductibilit√© : M√™me code = m√™me infra
- ‚úÖ S√©curit√© : Contr√¥le des mises √† jour
- ‚úÖ Rollback : Retour en arri√®re possible

### Contraintes de Version

```hcl

  source  = "terraform-aws-modules/vpc/aws"
  version = "5.1.2"
}

  source  = "terraform-aws-modules/vpc/aws"
  version = ">= 5.1.2"
}

# Compatible avec version (recommand√©)

  source  = "terraform-aws-modules/vpc/aws"

  source  = "terraform-aws-modules/vpc/aws"
  version = ">= 5.1.0, < 6.0.0"
}
```

### Semantic Versioning

```
Version: MAJOR.MINOR.PATCH (ex: 2.3.5)

MAJOR : Changements incompatibles (breaking changes)
MINOR : Nouvelles fonctionnalit√©s (r√©trocompatibles)
PATCH : Corrections de bugs

Exemples :

~> 2     ‚Üí  >= 2.0.0, < 3.0.0
```
